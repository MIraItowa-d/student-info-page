# 学信网网页模拟系统

一个模拟学信网的网页系统，采用图片热区+实际开发的混合方案。

## 📁 项目结构

```
学信网网页/
├── index.html              # 主HTML文件
├── request.md              # 需求文档
├── README.md               # 本说明文档
├── 页面1.jpg               # 首页背景图
├── 页面2.jpg               # 高等教育信息页背景图
├── 页面3.jpg               # 学籍详情参考图（上半部分）
├── 页面4.jpg               # 学籍详情参考图（下半部分）
├── css/
│   └── style.css          # 样式文件
├── js/
│   ├── router.js          # 路由控制
│   ├── form.js            # 表单逻辑
│   └── storage.js         # 数据存储
└── assets/
    └── default-avatar.jpg # 默认头像
```

## 🚀 快速开始

### 方式一：使用 Python 本地服务器（推荐）

```bash
# 在项目目录下运行
python3 -m http.server 8000

# 访问 http://localhost:8000
```

### 方式二：使用 Node.js

```bash
# 安装 serve
npm install -g serve

# 运行
serve

# 或使用 npx
npx serve
```

### 方式三：使用 VS Code Live Server

1. 安装 Live Server 插件
2. 右键 `index.html`，选择 "Open with Live Server"

## 📱 功能说明

### 页面1：学信档案首页
- **实现方式**：图片热区
- **可点击区域**：
  - ✅ "高等教育信息" → 跳转到页面2
  - ⏳ 其他区域 → 显示加载失败页面（永远加载不出来）

### 页面2：高等教育信息页
- **实现方式**：图片热区
- **可点击区域**：
  - ✅ "重庆邮电大学 本科" 学籍卡片 → 跳转到页面3
  - ◀ 返回按钮 → 返回首页

### 页面3：学籍详情页
- **实现方式**：完整开发
- **核心功能**：
  - 📷 上传/更换录取照片和学历照片
  - ✏️ 编辑所有学籍信息字段
  - 💾 数据保存到 LocalStorage
  - 🔄 刷新页面后数据保持

## 🎨 主要功能

### 1. 照片上传
- 点击照片区域或"录取照片"/"学历照片"标签
- 支持 jpg、png 等图片格式
- 自动压缩图片（最大宽度800px，质量80%）
- 限制文件大小5MB以内

### 2. 数据编辑
- 所有白色背景字段均可编辑
- 只读字段：学籍状态、预计毕业日期（灰色显示）
- 身份证号码格式验证

### 3. 数据持久化
- 使用 LocalStorage 存储
- 刷新页面后数据不丢失
- 照片以 Base64 格式存储

### 4. 表单验证
- 身份证号码：18位，最后一位可以是X
- 学制：1-10年
- 日期格式验证

## 🔧 技术栈

- **前端**：原生 HTML5 + CSS3 + JavaScript
- **存储**：LocalStorage
- **路由**：Hash 路由（#home, #education, #detail, #loading）
- **图片处理**：Canvas API 压缩

## 📝 使用说明

### 基础操作流程

1. 打开首页（页面1）
2. 点击"高等教育信息"进入页面2
3. 点击"重庆邮电大学 本科"卡片进入页面3
4. 在页面3编辑学籍信息：
   - 点击照片上传新照片
   - 修改各个字段信息
   - 点击"保存"按钮保存数据
5. 使用左上角返回按钮返回上一页

### 数据管理

**导出数据**（手动）：
```javascript
// 在浏览器控制台运行
console.log(localStorage.getItem('studentInfo'));
```

**清空数据**：
```javascript
// 在浏览器控制台运行
localStorage.removeItem('studentInfo');
location.reload();
```

**恢复默认数据**：
- 清空 LocalStorage 后刷新页面即可

## 🎯 注意事项

1. **图片文件完整性**
   - 确保 `页面1.jpg` 和 `页面2.jpg` 存在
   - 页面3和4.jpg仅作为参考

2. **浏览器兼容性**
   - 推荐使用 Chrome、Safari、Edge 最新版
   - 需支持 LocalStorage 和 FileReader API

3. **隐私保护**
   - 所有数据仅存储在本地浏览器
   - 不会上传到任何服务器
   - 清除浏览器数据会丢失保存的信息

4. **存储限制**
   - LocalStorage 上限约 5-10MB
   - 建议上传的照片不要过大

## 🐛 常见问题

### Q: 点击热区没有反应？
A: 确保使用本地服务器打开（file:// 协议可能有限制）

### Q: 照片上传后显示不出来？
A: 检查图片格式和大小，建议使用jpg/png，小于5MB

### Q: 保存后刷新页面数据丢失？
A: 检查浏览器是否允许 LocalStorage，或清除了浏览器数据

### Q: 页面2显示不出来？
A: 确认 `页面2.jpg` 文件存在且完整
